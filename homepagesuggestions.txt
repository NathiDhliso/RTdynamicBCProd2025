The Main Problems

Your video isn't optimized for scrubbing - Standard video encoding creates seek delays
Redundant animation loops - You have both a custom smooth scroll implementation AND complex ScrollTrigger animations competing for resources
Excessive filter animations - The multiple filter changes on the video element are performance killers

The Solution: Optimized Video + Simplified Animation
Here's a refined implementation that will give you truly smooth video scrubbing:Optimized HomePage with Smooth Video ScrubbingInteractive artifactÂ Let me provide you with the optimized code that will work properly. Here's what I've fixed:
Key Optimizations Made:

Removed the custom smooth scroll - It was competing with GSAP's ScrollTrigger
Simplified video animation - Used a single ScrollTrigger with scrub: 1 for smooth interpolation
Removed complex filter animations - They were killing performance; now just using opacity
Used useLayoutEffect for GSAP setup to prevent layout shifts
Proper cleanup with GSAP context - Prevents memory leaks
Simplified the overall animation structure - Less is more for performance

Critical Next Step: Video Optimization
The code alone won't make it "smooth as butter" - you MUST optimize your video file. Here's what you need to do:
Video Encoding Command (using FFmpeg):
bash# For MP4 (best compatibility)
ffmpeg -i Bar_Graph_Logo_Animation_Video.mp4 -vcodec libx264 -x264-params keyint=10:scenecut=0 -bf 0 -preset medium -crf 23 -acodec copy Bar_Graph_Logo_Animation_Video_optimized.mp4

# For WebM (better compression, modern browsers)
ffmpeg -i Bar_Graph_Logo_Animation_Video.mp4 -vcodec libvpx-vp9 -g 10 -b:v 2M -acodec copy Bar_Graph_Logo_Animation_Video.webm
What these commands do:

keyint=10 / -g 10: Forces a keyframe every 10 frames (crucial for smooth scrubbing)
scenecut=0: Disables automatic scene detection for consistent keyframes
-bf 0: Removes B-frames which slow down seeking

Additional Performance Tips:

Video Resolution: Keep it at 1920x1080 max for web. Higher resolutions will cause lag.
Frame Rate: 30fps is ideal. 60fps is unnecessary and doubles the decoding load.
